peko keys <add|list|remove> [OPTIONS]

Manages the per-project signing keys used to sign release builds. Key
files are copied into .peko/keys/<platform>/ and passwords are stored
in the operating system keychain. The add subcommand registers a
platform's key files and password, list shows the registered keys, and
remove deletes a platform's keys.

OPTIONS
    --platform=<PLATFORM>     target platform: android, ios, macos, or
                              windows (required for add and remove)
    --password=<VALUE>        password for the key file
    --password-file=<PATH>    read the key file password from a file
    --keystore=<PATH>         android: Java keystore file
    --alias=<NAME>            android: key alias in the keystore
                              (defaults to upload)
    --store-password=<VALUE>  android: keystore password (defaults to
                              --password)
    --key-password=<VALUE>    android: key password (defaults to
                              --password)
    --cert=<PATH>             ios and macos: signing certificate as a .p12
    --profile=<PATH>          ios: provisioning profile (.mobileprovision)
    --entitlements=<PATH>     ios and macos: entitlements plist (optional)
    --pfx=<PATH>              windows: code signing certificate as a .pfx

EXAMPLES
    peko keys add --platform=android --keystore=upload.keystore --alias=upload --password=secret
    peko keys add --platform=ios --cert=dist.p12 --profile=app.mobileprovision --password=secret
    peko keys add --platform=macos --cert=devid.p12 --password=secret
    peko keys add --platform=windows --pfx=codesign.pfx --password=secret
    peko keys list
    peko keys remove --platform=windows
